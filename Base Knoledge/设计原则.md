## 1. SOLID原则
| 原则   | 简称  | 核心思想        |
| ---- | --- | ----------- |
| 单一职责 | SRP | 一个类只做一件事    |
| 开闭原则 | OCP | 对扩展开放，对修改关闭 |
| 里氏替换 | LSP | 子类可透明替换父类   |
| 接口隔离 | ISP | 接口要小而专      |
| 依赖倒置 | DIP | 依赖抽象不依赖实现   |
## 2. 其他关键原则
- **DRY原则**  
  ▶ 不要重复代码  
  ▶ 通过抽象消除重复
- **KISS原则**  
  ▶ 保持简单直接  
  ▶ 避免过度设计
- **YAGNI原则**  
  ▶ 只实现当前需要的功能  
  ▶ 拒绝超前开发
